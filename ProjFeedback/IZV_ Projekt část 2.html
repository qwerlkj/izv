Funkce isin

Pro testování více podmínek najednou se často objevovalo řešení na následujícím principu. Všímaví asi poznají, že podmínku pro rok lze vytknout

df_regs = df[((df["region"] == regions[0]) & (df['date'].dt.year < 2021)) |
             ((df["region"] == regions[1]) & (df['date'].dt.year < 2021)) |
             ((df["region"] == regions[2]) & (df['date'].dt.year < 2021)) |
             ((df["region"] == regions[3]) & (df['date'].dt.year < 2021))].copy()

Ovšem z pohledu kódu se nejedná o nejlepší (univerzální) řešení. Při přidání dalšího regionu se musí přepisovat kód. Proto je lepší použít následucí konstrukci.

df_regs = df[df["region"].isin(regions) & (df['date'].dt.year < 2021)].copy()

Objevilo se i řešení využivající query, které může být elegantní pro rychlé testování. Někdy se však trochu komplikuje

    regions = {"OLK": 0, "MSK": 1, "JHM": 2, "ULK": 3}  
    cond = "| ".join(4*["region == '{}' "])
    cond = f"({cond}) & (p58 == 5)"
    df = df.query(cond.format(*(regions.keys()))).copy()

a je jednodušší se mu buď vyhnout

    df = df[df["region"].isin(regions.keys()) & (df["p58"] == 5)]

nebo využít @ notaci pro získání objektu existující v aktuálním scope.

    regs = list(regions.keys())
    df = df.query("region in @regs and p58 == 5")

Využití funkce tight_layout()

Často se stávalo, že legenda překrývala graf, popisky a titulky zasahovaly mimo kreslící plochu či do ostatních podgrafů atd. To řeší funkce plt.tight_layout() nebo g.fig.tight_layout(). Pokud používáme legendu nebo nadpis suptitle, tak je vhodné přidat parametr rect = [0, 0, 1, 1] (který značí relativní začátek a konec oblastí, kam se kreslí).

Vizuální a jazyková úroveň

Když vizualizujete data, je nutné připravit rozumný pohled pro čtenáře. Proto by všechny labely měly mít stejnou velikost, titulky by měly být větší a nadřazený titulek grafu největší. Je vhodné si pohrát s poměrem stran (aspect). V těchto případech čtvercové grafy nevypadaly nejlépe.

Další kapitolou je jazyk popisků. Měl by být spisovný, s diakritikou, rozumně volená velká/malá písmena. Opravdu vypadá zvláště, když mixujete např. slovenštinu s angličtinou.

Procházení dat

Zabijákem výkonnosti je procházení všech dat iterativně. Zejména pokud pracujeme s číselnými hodnotami.

def _convert_to_causes(row):
    if row["p10"] in [1, 2]:
        return "řidičem"
    elif row["p10"] == 4:
        return "zvěří"
    else:
        return "jiné"

df["p10_new"] = df.apply(lambda row: _convert_to_causes(row), axis=1)
# poznamka: lambda funkce je zbytecna
# df["p10_new"] = df.apply(_convert_to_causes, axis=1)

# 5.38 s ± 23.1 ms per loop (mean ± std. dev. of 7 runs, 1 loop each)

Pokud bychom pracovali pouze s jednou sérií (ne celým DataFrame), tak ušetříme přeskládávání pár set megabytů v paměti (viz velikost datasetu z prvního úkolu).

def _convert_to_causes(c):
    if c in [1, 2]:
        return "řidičem"
    elif c == 4:
        return "zvěří"
    else:
        return "jiné"

df["p10_new"] = df["p10"].apply(_convert_to_causes)
# 53.2 ms ± 509 µs per loop (mean ± std. dev. of 7 runs, 10 loops each)

Můžeme i logiku dotazů otočit a omezit další testování.

df["new_p10"] = "jiné"
df.loc[df["p10"] == 4, "new_p10"] = "zvěří"
df.loc[df["p10"].isin([1, 2]), "new_p10"] = "řidičem"
# 26.3 ms ± 52.6 µs per loop (mean ± std. dev. of 7 runs, 10 loops each)

Rozdíl časů: celkový rozdíl operace nad datasetem je 5.38 sekund versus 26.3 ms!!! Časy byly počítány pro celý rozsah. Mezi druhou a třetí implementací už není tak výrazný rozdíl, protože Pandas pracuje se str objekty jako s PyObject.

Drobné zrychlení (za cenu toho, že musíme vyjmenovat všechny možnosti) nám přinese funkce replace.

df["p10_new"] = df["p10"].replace({
       **{1: "řidičem", 2: "řidičem", 4: "zvěří"}, 
       **{i: "jiné" for i in [3, 5, 6, 7, 0]} # pro jednoduchost generátor
    })
# 17.3 ms ± 336 µs per loop (mean ± std. dev. of 7 runs, 100 loops each)

Vytváření zbytečných funkcí

Často se vyskytovaly případy, kdy někdo přepisoval základní funkce Pandas do svých implementací. Zejména u úplně základních funkcí to bylo zbytečné.

def _remove_unwanted_data(df: pd.DataFrame, df_new: pd.DataFrame, data_to_keep: list):
    for column in df:
        if column not in data_to_keep:
            df_new.drop(columns=column, inplace=True)

def _remove_rows(df: pd.DataFrame, index: str, rows: list):
    df.set_index(index, inplace=True)
    df.drop(rows, axis=0, inplace=True)
    df.reset_index(inplace=True)

_remove_unwanted_data(df, df_conditions, ["p18", "region", "Date"])
_remove_rows(df_conditions, "region", ['HKK', 'JHC', 'JHM', 'KVK', 'MSK', 'PAK', 'PHA', 'PLK', 'STC', 'ULK'])

Při výběru dat přes indexaci [] je možné přímo získat položky, které potřebuji (sloupce nebo boolean podmínkou). Problém je však to, že Google jako první nabídne DataFrame.drop(), což není nejefektivnější.

Potom se dá výběr zjednodušit na dva (případně na jeden) výběry. Otázka je, jestli je výhodné vyjmenovat všechny položky, co nechceme (při výběru musíme negovat ~), nebo jen ty čtyři, které chceme.

# existuji na to primo funkce
df_conditions = df[["p18", "region", "Date"]]
df_conditions = df[~df["region"].isin(['HKK', 'JHC', 'JHM', 'KVK', 'MSK', 'PAK', 'PHA', 'PLK', 'STC', 'ULK'])]

# nebo v jednom lokatoru
df_conditions = df.loc[
         ~df["region"].isin(['HKK', 'JHC', 'JHM', 'KVK', 'MSK', 'PAK', 'PHA', 'PLK', 'STC', 'ULK']), 
         ["p18", "region", "Date"]
        ]

Ukázka řešení

Přidávám i ukázku, jak nám využití funkcí seaborn a pandas mohlo ulehčit práci a mohli jsme jednoduše zobrazit graf povětrnostních podmínek.

def plot_conditions(df: pd.DataFrame, fig_location: str = None,
                    show_figure: bool = False):
    df2 = df.query(
        "region in ['JHM', 'MSK', 'OLK', 'ZLK'] and p18 != 0").copy()

    # Prejmenovani integeru na textovou reprezentaci
    df2["Podmínky"] = df2["p18"].replace({
        0: "jiné",
        1: "neztížené",
        2: "mlha",
        3: "na počátku deště",
        4: "déšť",
        5: "sněžení",
        6: "náledí",
        7: "vítr"
    })

    # Otoceni tabulky do "sirokeho" format
    df2 = df2.pivot_table(index=["date", "region"], columns=["Podmínky"], values=[
                          "p1"], aggfunc="count").reset_index()
    
    # Shrnuti na mesice podle kazde podminky
    df2 = df2.groupby("region").resample("1M", on="date").agg(
        "sum").reset_index().melt(["date", "region"])

    # Vytvoreni relacniho grafu (figure-level pristup)
    g = sns.relplot(kind="line", data=df2,
                    x="date", y="value", hue="Podmínky", col="region",
                    col_wrap=2, height=3, aspect=1.5, facet_kws=dict(sharex=False),
                    )

    # Nastaveni okraju do 1. 1. 2021 (filtrace roku 2021 pryc)
    g.set(xlim=np.array(["2016-01-01", "2021-01-01"], dtype=df2["date"].dtype),
          ylim=(0, None),
          xlabel=None)

    g.set_titles("Kraj: {col_name}")

    # Kazdemu grafu nastavime popisky x-ove osy spravne
    for ax in g.axes.ravel():
        xticks = ax.get_xticks()
        ax.set_xticks(xticks)
        ax.set_xticklabels(
            [pd.to_datetime(tm, unit='d').strftime(r'%m/%y') for tm in xticks])
    for i in [0, 2]: # a take popis y osy (jen u levych grafu)
        g.axes[i].set_ylabel("Počet nehod")

    # Doladeni layoutu a ulozeni
    g.fig.tight_layout(rect=(0, 0, .85, 1))
    if fig_location:
        g.fig.savefig(fig_location)
    if show_figure:
        g.fig.show()
